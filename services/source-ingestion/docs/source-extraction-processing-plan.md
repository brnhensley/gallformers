# Source Upload and Extraction:

## Entry Point and File Processing

There will be 1 entry point. A screen to either upload a PDF and process it or to select a previously processed file. The list of previously processed files will be shown in a sortable table showing: the extracted title as a clickable link, status (TBD exactly, but something like, source mapped, x of y species processed). 

Upon upload of a PDF, a hash of the file computed. If the hash matches an already processed file then processig will be skipped and the user taken to the page for that file.

The processing pipeline will:
1. extract - Python text extraction of the PDF
2. preprocess - Python heuristic to clean the text up, remove boilerplate (page numbers, journal titles, etc), rejoin split words, etc.
3. llm-clean w/ deepinfra/deepseek-ai/DeepSeek-V3 - LLM clean up of extraction errors, misspellings, etc.
4. metadata w/ deepinfra/deepseek-ai/DeepSeek-V3 - extract metadata, title, author, etc.
5. data-extract w/ deepinfra/deepseek-ai/DeepSeek-V3 - this is the big one, pulls species, traits, etc.
6. assemble - Python to post-process all the results

The pipeline will store all intermiediate and final artifacts as json and markdown for future work/processing. This prevents the need to re-process once processed, as well as beign able to resume failed processing from the last successful step.

## Processed File Screen

Once a file is processed a details screen will be available. The screen will be driven by the stored artifacts generated by the pipeline. It will show:

The Source and any match to an existing Source as well as a way to create the Source if it does not already exist

A list of gall species that were found as well as their processing status. Clicking on one takes you into a modal view where all of the details are shown as well as a way to map to an existing species or add a new one. The details shown are:

- The extracted gall species name
- All hosts that were extracted, mapped or existing hosts or a new/lookup mechanism to map or create
- Gall traits
- The descriptive text related to the gall
- The entire text of the article in an expandable text screen

When a gall is mapped/added it will include the mapping to the Source.

## Processing Status

Once we have a file processed by the pipeline a user can further process it. This process is the act of mapping to, updating, and/or creating galls and their related data in the primary database. The following items will be considered as part of the processing status of a file:

- Source - has it been mapped or added to the DB?
- Galls - for each one has it been mapped or added to the DB?
